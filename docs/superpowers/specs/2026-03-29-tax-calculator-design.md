# 役員報酬最適化シミュレーター 設計書

v2 — 11ワーカーによる調査・検証・レビューを経て統合

## 概要

一人法人（中小企業）の役員報酬と法人利益の配分を視覚的にシミュレーションし、手取り・税負担の最適化を支援するWebアプリケーション。

**市場ポジション**: 事前確定届出給与の社保最適化をセルフサービスで提供するツールは市場に存在しない。税理士事務所の対面シミュレーション（有料）の精度を、無料のWebツールで実現する。

## 対象ユーザー

小規模法人（資本金1億円以下）の経営者・役員。汎用ツールとして公開し、個人プリセットはlocalStorageで管理（ランタイム読込、ビルド成果物に含めない）。

## 技術スタック

- **フレームワーク**: Nuxt 3（SSG）
- **UIライブラリ**: shadcn-vue（Radix Vue ベース）
- **スタイル**: Tailwind CSS、ダーク基調
- **パッケージマネージャ**: pnpm
- **バリデーション**: Zod（localStorage読込時の型安全検証）
- **デプロイ**: Cloudflare Pages
- **データ永続化**: localStorage（シナリオ保存）

## アーキテクチャ

### 3層分離

計算ロジック・リアクティビティ・データ定義を明確に分離する。

```
utils/                          # 純粋関数のみ（Vue依存なし、テスト容易）
  tax-calculator.ts             # 所得税計算
  resident-tax-calculator.ts    # 住民税計算
  corporate-tax-calculator.ts   # 法人税等計算
  social-insurance-calculator.ts # 社保計算
  deduction-calculator.ts       # 控除計算
  optimizer.ts                  # 最適化アルゴリズム
  rounding.ts                   # 端数処理関数（集約）

composables/                    # Vueリアクティビティのみ
  useTaxEngine.ts               # 統合層（computed chain → utils呼出）
  useScenario.ts                # シナリオ保存/読込（localStorage + Zod検証）

config/                         # データ定義のみ（関数なし）
  tax-rates/
    types.ts                    # 共通型定義（satisfiesで型完全性保証）
    index.ts                    # 年度レジストリ
    2025.ts                     # 令和7年分の所得税・住民税テーブル
  corporate-tax/
    2025.ts                     # 令和7年度の法人税等テーブル
  social-insurance/
    2025.ts                     # 令和7年度の社保料率・等級テーブル（全50等級）
  deductions.ts                 # 控除定義（所得税/住民税の差異を含む）
  warnings.ts                   # 警告条件定義
```

### リアクティブデータフロー

すべて `computed` チェーンで表現。`watch` は副作用（localStorage保存）のみに使用。

```
monthlyCompensation (ref)  ───┐
bonusAmount (ref)          ───┤
                               ├→ socialInsurance (computed)
                               │     ├→ personalDeductions (computed)
deductionSettings (ref)    ───┤     │     ├→ incomeTax (computed)
                               │     │     └→ residentTax (computed)
totalProfit (ref)          ───┼─────┤
                               │     └→ corporateTax (computed)
                               │
                               └→ summary (computed) → UI描画
```

### パフォーマンス

- 1回の全計算: 約0.01ms（数百回の算術演算）。debounce不要
- スライダー操作時: `requestAnimationFrame` でスロットリング（描画と同期）
- 最適化（ブルートフォース）: 最大5,000パターンでも0.05ms。Web Worker不要
- 社保テーブル検索: 二分探索に変更（50等級の線形探索でも問題ないが意図が明確）

## UI設計

### レイアウト

**2カラム構成**（デスクトップ）→ **1カラム**（モバイル、レスポンシブ）

- **左カラム（入力パネル）**: 基本設定、報酬配分スライダー、控除設定、最適化ゴール
- **右カラム（結果パネル）**: ヒーローナンバー（sticky）、配分フロー、税負担内訳、ブラケット可視化、シナリオ比較

### ヒーローナンバー（sticky）

右カラム最上部に固定。スクロールに追従し、スライダー操作中も常に結果が見える。

```
┌──────────────────────────────────────┐
│        トータル手残り                 │
│       ¥14,230,000  (+12.3万)         │
│                                      │
│  個人手取り ¥8.42M  法人留保 ¥5.81M   │
└──────────────────────────────────────┘
```

差額はスライダー操作時にカウントアップ/ダウンアニメーションで表示（200msイージング）。

### スライダー設計

**マスタースライダー + 内訳配分**

1. 「年間報酬総額」マスタースライダー（法人に残す額が直感的に分かる）
2. 内訳: 月額 / 事前確定届出給与の比率サブスライダー
3. **数値直接入力を必ず併用**（万円単位、スライダーと双方向バインド）
4. モバイル: ステッパーボタン（-/+）、数値タップで直接入力モード
5. 主要ブラケット境界にスナップポイント（任意ON/OFF）

制約バー: 利益総額をフルバーとして、月額（青）・賞与（紫）・社保会社負担（灰）・法人利益（緑）をスタックドバーで可視化。

### 控除設定の段階的開示（3層）

```
Layer 1: 自動適用（操作不要・表示のみ）
  基礎控除 + 給与所得控除 + 社保控除
  → まとめて「自動控除: ¥2,430,000」と1行表示。展開で内訳。

Layer 2: よく使う控除（トグルのみ）
  配偶者控除、扶養控除、小規模企業共済、iDeCo
  → カード形式でON/OFF。ONで金額表示。

Layer 3: その他の控除（折り畳み内）
  生命保険料控除、医療費控除、その他
  → 「その他の控除を追加 (+3)」ボタンで展開。
```

### 警告の3レベル階層化

| レベル | 表示方法 | 対象 |
|---|---|---|
| **Critical**（赤） | 全幅バナー、無視不可 | 赤字警告、入力エラー |
| **Warning**（黄） | スライダー横にバッジ、クリックで詳細 | 過大/過少報酬、社保資格喪失、事前確定届出リスク |
| **Info**（アイコン） | 該当数値横にツールチップ | 厚年上限、基礎控除減額、確定申告要否 |

一度に表示する警告は最大3件。超過分は「他N件の注意事項」で折り畳み。

### ブラケット可視化

**2段階表示**（所得税・法人税の両方に適用）:
1. **横バー（常時表示）**: 各ブラケットを色分け。現在位置を光るマーカーで表示。次のブラケットまでの余裕額を数字で表示。
2. **縦バー（展開時）**: 各ブラケットの税額を棒グラフ + 説明テキスト。実効税率と限界税率の違いを説明。

### 配分フロー可視化

フロー型スタックドバーで「利益がどこに流れるか」を表示:

```
利益総額 ¥20,000,000
  ├── 個人へ: 手取り / 所得税 / 住民税 / 社保(個人)
  └── 法人に残す: 内部留保 / 法人税等 / 社保(会社)
```

### デザイントーン

ダーク基調・Fintech風。shadcn/uiダークテーマベース。

**色覚多様性対応**: 色だけに頼らない。増減は +/- プレフィックス + 矢印を併用。ブラケットバーには色に加えてラベルを付ける。カラーパレットは WCAG 2.1 AA基準（コントラスト比4.5:1以上）。色はCSS変数で管理（将来のライトテーマ対応）。

### 数値フォーマット

- ヒーローナンバー: `1,423万円`（万円表示）
- 内訳テーブル: `14,230,000円`（円表示）
- スライダー: `月額 83万円`
- ヘッダーに `万円 | 円` トグル

### ユーザーフロー

1. 利益総額を入力（初回はガイドテキスト付き、右カラムにサンプルデータ表示）
2. プロフィール設定（年齢・都道府県）
3. 最適化ゴール選択 → 自動最適値算出（理由の1文説明付き）
4. スライダー/数値入力で自由に探索（リアルタイム更新）
5. シナリオとして保存（自動命名: `月額80万+賞与200万_手取り842万`、編集可能）
6. 複数シナリオ比較（タブ切替、左に一覧・右に差分テーブル）

## 入力パラメータ

| パラメータ | 説明 | デフォルト |

| 利益総額 | 役員報酬・社保会社負担控除前の利益（= 売上 - 役員報酬以外の全経費） | 20,000,000円 |

| パラメータ | 説明 | デフォルト |

| 都道府県 | 協会けんぽ料率の決定 | 東京都 |
| 資本金 | 均等割・税率判定 | 100万円 |
| 法人所在地 | 超過税率の適用判定 | 東京23区 |

### 控除設定

**トグル選択 → 自動金額入力 → 手動上書き可能**

| 控除 | 自動入力値 | 備考 |
|---|---|---|
| 基礎控除 | 所得に応じて自動（所得税: R7特例6段階、住民税: 43万） | 所得税/住民税で金額が異なる。年度テーブル別管理 |
| 給与所得控除 | 給与収入から自動計算 | 令和7年〜: 最低65万、上限195万 |
| 社会保険料控除 | 社保個人負担分を全額自動反映 | — |
| 所得金額調整控除 | 23歳未満扶養ありで自動 | (給与-850万)×10%、上限15万。所得税のみ |

| 生命保険料控除 | 新制度: 所得税最大12万 / 住民税最大7万 | **所得税と住民税で計算テーブルが異なる**（別々に実装） |
| 医療費控除 | (医療費 - 10万)、上限200万 | 金額入力 |
| その他控除 | 合計額を自由入力 | — |

## 計算ロジック

### 端数処理ルール（集約）

すべての端数処理を `utils/rounding.ts` に集約し、各計算パスで一貫して適用する。

```typescript
// 課税所得: 1,000円未満切捨て（所得税・法人税共通）
roundTaxableIncome(n) = Math.floor(n / 1_000) * 1_000

// 税額: 100円未満切捨て（所得税+復興税合計、法人税、地方法人税、法人事業税、特別法人事業税、住民税所得割）
roundTaxAmount(n) = Math.floor(n / 100) * 100

// 社保保険料: 50銭以下切捨て、50銭超切上げ
roundInsurancePremium(n) = n - Math.floor(n) <= 0.5 ? Math.floor(n) : Math.ceil(n)

// 標準賞与額: 1,000円未満切捨て
roundStandardBonus(n) = Math.floor(n / 1_000) * 1_000

// 料率適用: 乗算直後に即座に丸める（浮動小数点誤差対策）
applyRate(base, rate, rounding) = rounding(base * rate)
```

### Step 1: 社会保険料の計算

#### 1-a. 月額報酬の社保

```
月額報酬 → 標準報酬月額等級テーブル参照（二分探索）
  ├ 健康保険: 1〜50等級（上限1,390,000円）
  └ 厚生年金: 1〜32等級（上限650,000円）
  ※ 健保1-3級は厚年第1級(88,000)、健保36-50級は厚年第32級(650,000)

健康保険料（月額）= applyRate(標準報酬月額(健保), 健保料率, roundInsurancePremium)
  └ 40歳以上65歳未満: 健保9.91% + 介護1.59% = 11.50%
  └ 40歳未満 or 65歳以上: 健保9.91%のみ
  → 労使折半（各50銭処理）

厚生年金保険料（月額）= applyRate(標準報酬月額(厚年), 18.3%, roundInsurancePremium)
  → 労使折半

子ども・子育て拠出金（月額）= applyRate(標準報酬月額(厚年), 0.36%, Math.floor)
  → 全額事業主負担

各月額 × 12 → 年間保険料
```

#### 1-b. 賞与の社保（支給回数ごとに個別計算）

```
事前確定届出給与の支給回数に応じて、各回の金額を個別に計算:

各回の賞与額 → 標準賞与額（1,000円未満切捨て）
  ├ 健康保険: min(標準賞与額, 年度累計573万 - これまでの累計)
  └ 厚生年金: min(標準賞与額, 1回あたり150万)
  ※ 厚年上限は「1回あたり」で判定。年2回300万なら各回150万上限=合計300万対象
  ※ 健保上限は「年度累計」で判定。年2回各300万なら1回目300万+2回目273万=累計573万

保険料率は月額と同一
子ども・子育て拠出金も賞与に適用（厚年と同じ上限）
```

#### 1-c. 社保合計

```
個人負担 = 月額分個人負担 + 各回賞与分個人負担の合計
会社負担 = 月額分会社負担 + 各回賞与分会社負担の合計 + 拠出金合計
社保総コスト = 個人負担 + 会社負担
```

### Step 2: 個人の税金計算

#### 2-a. 給与所得

```
給与収入 = 年間役員報酬
給与所得控除 = テーブル参照（令和7年分〜）
  ├ 〜190万: 65万円（定額）
  ├ 〜360万: 収入×30% + 8万
  ├ 〜660万: 収入×20% + 44万
  ├ 〜850万: 収入×10% + 110万
  └ 850万超: 195万（上限）
給与所得 = 給与収入 - 給与所得控除
```

#### 2-b. 所得控除（所得税用 / 住民税用で別計算）

```
所得控除（所得税用）:
  社会保険料控除（全額）
  + 基礎控除（R7特例6段階: 132万以下→95万 ... 655万超~2,350万以下→58万）
  + 所得金額調整控除（給与850万超+23歳未満扶養: (給与-850万)×10%, 上限15万）
  + 各種控除

所得控除（住民税用）:
  社会保険料控除（全額）
  + 基礎控除（R7: 43万固定 ※2,400万以下。R8〜: 改正予定、年度テーブルで管理）
  + 各種控除（住民税固有の金額を使用、特に生命保険料控除は計算テーブルが異なる）
```

#### 2-c. 課税所得

```
所得税の課税所得 = roundTaxableIncome(給与所得 - 所得控除(所得税用))
住民税の課税所得 = roundTaxableIncome(給与所得 - 所得控除(住民税用))
※ 0未満は0
```

#### 2-d. 所得税

```
速算表:
  〜195万:  5%
  〜330万: 10% - 97,500
  〜695万: 20% - 427,500
  〜900万: 23% - 636,000
  〜1,800万: 33% - 1,536,000
  〜4,000万: 40% - 2,796,000
  4,000万超: 45% - 4,796,000

復興特別所得税 = Math.floor(所得税額 × 2.1%)
所得税合計 = roundTaxAmount(所得税額 + 復興特別所得税)
※ 合計に対して100円未満切捨て
```

#### 2-e. 住民税

```
所得割 = roundTaxAmount(住民税の課税所得 × 10% - 調整控除)
均等割 = 5,000円（特別区民税3,000 + 都民税1,000 + 森林環境税1,000）
住民税合計 = 所得割 + 均等割

調整控除:
  課税所得200万以下: min(人的控除差額合計, 課税所得) × 5%
  課税所得200万超:   max(人的控除差額合計 - (課税所得 - 200万), 2,500) × 5%
  合計所得2,500万超: 0
```

### Step 3: 法人の税金計算

```
法人所得 = roundTaxableIncome(利益総額 - 年間役員報酬 - 社保会社負担合計)

法人税:
  800万以下: 15%（中小法人軽減税率）
  800万超:   23.2%
  → roundTaxAmount(合計)

地方法人税 = roundTaxAmount(法人税額 × 10.3%)

法人住民税:
  法人税割 = roundTaxAmount(法人税額 × 7.0%（標準）or 10.4%（超過）)
    ※ 超過: 法人税額年1,000万超 or 資本金1億超
  均等割 = 70,000円/年（資本金1,000万以下・従業者50人以下）

法人事業税（所得割）:
  400万以下:   3.5%（標準）/ 3.75%（超過）
  400〜800万:  5.3%（標準）/ 5.665%（超過）
  800万超:     7.0%（標準）/ 7.48%（超過）
  ※ 超過: 資本金1億超 or 年所得2,500万超
  → roundTaxAmount(合計)

特別法人事業税 = roundTaxAmount(基準法人所得割額（標準税率計算）× 37%)
  ※ 超過税率適用法人でも標準税率で計算

法人実効税率（損金算入効果反映）:
  = (法人税率 × (1 + 地方法人税率 + 住民税率) + 事業税率 × (1 + 特別法人事業税率))
    / (1 + 事業税率 × (1 + 特別法人事業税率))
  ※ 事業税が翌年度の損金になる効果を分母に反映
```

### Step 4: 結果集計

```
個人手取り = 年間役員報酬 - 社保個人負担 - 所得税合計 - 住民税合計
法人内部留保 = 法人所得 - 法人税等合計
トータル手残り = 個人手取り + 法人内部留保

追加指標:
  個人実効税率 = (所得税 + 住民税) / 年間役員報酬
  法人実効税率 = 上記の損金算入効果反映済み算式
  総合税負担率 = (全税金 + 全社保) / 利益総額
  社保負担率 = 社保総コスト / 利益総額
  現在の所得税ブラケット / 次のブラケットまでの余裕額

不変条件（テストで検証）:
  個人手取り + 法人内部留保 + 全税金 + 全社保 = 利益総額
```

### 最適化アルゴリズム

**2段階探索 + 支給回数の考慮**

```
Phase 1: 粗い探索（1万刻み）
  月額: 0, 1万, 2万, ..., min(利益総額/12, 200万)
  賞与: 0, 10万, 20万, ..., 利益総額 - 月額×12 - 推定社保
  支給回数: 1, 2, 3
  制約: 月額×12 + 賞与 + 社保会社負担 <= 利益総額
  → 各ゴール関数の上位5候補を特定

Phase 2: 精密探索（1,000円刻み）
  上位5候補の周辺 ±10万円 を1,000円刻みで再探索
  → 最適解を確定

計算量: Phase 1 ~5,000パターン + Phase 2 ~5,000パターン = ~10,000パターン ≈ 0.1ms
```

**最適化時の追加表示**:
- 「月額XX万がベストです。所得税のXX%ブラケット境界を活用しています」等の理由説明を自動生成
- 社保最小化ゴール選択時: 「将来の厚生年金受給額が概算で年間XX万円減少します」を併記

## 警告・バリデーション

### エラー（計算不可）

- 年間役員報酬 + 社保会社負担 > 利益総額 → 赤字警告
- 月額報酬 < 0 or 事前確定届出給与 < 0 → 入力エラー

### 警告（計算可能だが注意が必要）

| 条件 | メッセージ | レベル |
|---|---|---|
| 月額 = 0円 | 社会保険の加入資格を失う可能性があります | Warning |
| 事前確定届出給与のみ | 定期同額給与がない場合の実務リスクがあります | Warning |
| 報酬 > 利益の50% | 過大役員報酬と認定されるリスクがあります（税務調査注意） | Warning |
| 報酬 < 利益の10% | 極端に低い報酬は行為計算否認のリスクがあります | Warning |
| 前年比20%超の報酬変動 | 大幅な変更は税務調査で理由を問われる可能性があります | Warning |
| 月額 > 厚年上限(65万) | 厚生年金保険料は上限に達しています | Info |
| 等級境界付近 | 1万円の変動で年間社保がXX万円変わります | Info |
| 賞与 > 150万 | 厚生年金の標準賞与額上限超過（年金受給額への影響あり） | Info |
| 賞与 > 573万 | 健康保険の標準賞与額年間上限超過 | Info |
| 合計所得 > 2,400万 | 基礎控除が減額されます | Info |
| 事前確定届出の支給回数 >= 4 | 年4回以上は報酬として社保計算に含まれます（入力制限: 1-3回） | Warning |
| 年間報酬 > 2,000万 | 確定申告が必要です。税理士への相談を推奨します | Info |
| 法人所得 = 0 | 赤字でも均等割70,000円は課税されます | Info |
| 社保最小化ゴール選択中 | 将来の年金受給額が概算で年間XX万円減少します | Warning |
| 法人に高額な内部留保 | 法人資金の個人使用は役員貸付金として認定利息が課税されます | Info |

### 免責表示

フッターに「免責事項」リンク（モーダル表示）。結果パネル下に1行の概算注記を常時表示。印刷時にフルテキスト出力。

- 本シミュレーションの結果は概算であり、実際の税額を保証するものではありません
- 税制は毎年改正される可能性があります（適用税率: 令和7年度）
- 税務署による過大役員報酬の認定リスク、同族会社の行為計算否認リスクは考慮していません
- 重要な経営判断の際は税理士にご相談ください
- 事前確定届出給与は届出と1円でも異なる金額を支給した場合、**全額が損金不算入**になります（届出期限: 株主総会から1か月以内 or 事業年度開始から4か月以内の早い方）

## テスト戦略

### ゴールデンテスト

国税庁の計算例・協会けんぽの保険料額表から抽出した既知の正解値で検証。

### プロパティベーステスト

計算結果の不変条件を検証:
- `個人手取り + 法人内部留保 + 全税金 + 全社保 = 利益総額`（閉じた系の保存則）
- 全金額が非負
- 報酬増加 → 個人税負担は単調増加
- 報酬増加 → 法人内部留保は単調減少

### スナップショットテスト

税率テーブルの意図しない変更を検出。テーブル更新時はスナップショット再生成 + レビュー必須。

## 前提条件・制約

- 一人法人（代表取締役のみ）を主な想定
- 協会けんぽ加入（健保組合は未対応）
- 東京都特別区をデフォルト（都道府県切替可能）
- 定常状態の年次計算（初年度特有のズレは考慮しない）
- 雇用保険・労災保険は対象外（役員は原則適用外）
- 消費税は考慮しない
- 他の所得（配当、不動産等）は考慮しない
- 事前確定届出給与の支給回数は1-3回に制限（4回以上は報酬扱い）
